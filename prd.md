# PRD: Система управления рисками организации (СУР)

## 1. Общая информация

| Поле | Значение |
|---|---|
| Документ | Product Requirements Document (PRD) |
| Продукт | Система управления рисками организации (СУР) |
| Заказчик | Риск-менеджер организации |
| Методологическая база | ISO 31000:2018, COSO ERM 2017 |
| Формат реализации | Книга Excel (.xlsx) с формулами и условным форматированием |
| Статус | v1.0 — реализовано |

## 2. Цель и контекст

Разработать инструмент управления рисками организации, который позволяет риск-менеджеру:
- вести единый реестр рисков;
- планировать и отслеживать мероприятия по обработке рисков;
- визуализировать риски на тепловой карте (матрице) для приоритизации и отчётности перед руководством.

Система должна соответствовать процессу управления рисками по **ISO 31000:2018** (установление контекста, идентификация, анализ, оценивание, обработка, мониторинг и пересмотр, коммуникация и консультирование) и компонентам **COSO ERM 2017** (Governance & Culture, Strategy & Objective-Setting, Performance, Review & Revision, Information/Communication/Reporting).

## 3. Пользователи

- **Риск-менеджер** — основной пользователь, ведёт реестр, формирует отчётность.
- **Владельцы рисков** — руководители подразделений, ответственные за конкретные риски и мероприятия.
- **Руководство / Комитет по рискам** — потребители отчётности (тепловая карта, сводки).

## 4. Функциональные требования

### 4.1 Лист «Инструкция»
- Описание состава книги и назначения каждого листа.
- Явное сопоставление функциональности с этапами процесса ISO 31000.
- Явное сопоставление функциональности с компонентами COSO ERM.
- Пошаговый порядок работы с книгой (5 шагов: настройка справочников → заполнение реестра → создание плана обработки → использование матрицы → регулярный пересмотр).

### 4.2 Лист «Справочники»
Обязательные шкалы и списки, используемые для стандартизации данных и выпадающих списков (data validation):
- **Шкала вероятности** (1–5): Редко / Маловероятно / Возможно / Вероятно / Практически неизбежно — с текстовым описанием частоты.
- **Шкала влияния** (1–5): Незначительное / Низкое / Умеренное / Высокое / Критическое — с описанием финансового/операционного/репутационного эффекта.
- **Шкала уровня риска (5 цветов)** и риск-аппетит: диапазоны баллов 1–4, 5–9, 10–14, 15–19, 20–25 с рекомендуемым реагированием для каждой зоны.
- Справочники: категории риска (8 шт.), стратегии реагирования (4 шт.: избежание/снижение/передача/принятие), статусы риска (4 шт.), эффективность контролей (4 шт.), статусы мероприятий плана обработки (5 шт.).

### 4.3 Лист «Реестр рисков»
Обязательные поля (21 колонка):
1. ID (автогенерация по заполнению наименования риска).
2. Категория (выпадающий список).
3. Наименование риска.
4. Причина возникновения.
5. Последствия / влияние.
6. Владелец риска.
7. Дата выявления.
8–9. Вероятность и Влияние **присущего** (inherent) риска (1–5, выпадающий список).
10. Уровень риска (присущий) = Вероятность × Влияние (автоформула).
11. Зона (присущий) — автоматическое определение по баллу, с условным форматированием в 5 цветов.
12. Существующие меры контроля.
13. Эффективность контролей (выпадающий список).
14–15. Вероятность и Влияние **остаточного** (residual) риска.
16. Уровень риска (остаточный) — автоформула.
17. Зона (остаточный) — автоформула + условное форматирование.
18. Стратегия реагирования (выпадающий список).
19. Статус риска (выпадающий список).
20. Дата следующего пересмотра.
21. ID плана обработки (связь с листом плана).

Требования к поведению:
- Автоматический расчёт уровня риска по формуле, без ручного ввода баллов.
- Условное форматирование ячеек баллов и текстовых зон в 5 цветов согласно шкале риск-аппетита.
- Валидация ввода (выпадающие списки) для категорий, оценок, стратегий, статусов и эффективности контролей.
- Заполнено 7 примеров рисков разных категорий (операционный, финансовый, комплаенс, HR, ИБ, стратегический, проектный) для демонстрации.

### 4.4 Лист «План обработки рисков»
Обязательные поля (14 колонок):
- ID мероприятия (автогенерация).
- ID риска (выпадающий список из Реестра рисков).
- Наименование риска — автоматически подтягивается (VLOOKUP) из Реестра.
- Стратегия реагирования — автоматически подтягивается (VLOOKUP) из Реестра.
- Мероприятие / действие, Ответственный, Ресурсы/бюджет.
- Сроки начала и завершения.
- KRI / индикатор контроля.
- Целевой остаточный уровень риска.
- Статус выполнения (выпадающий список) и % выполнения.
- Комментарии.

Требования к поведению:
- Связь «многие ко многим» с Реестром рисков через ID риска.
- Условное форматирование % выполнения (зелёный ≥100%, жёлтый 50–99%, оранжевый <50%).
- Выделение просроченных мероприятий красным текстом.

### 4.5 Лист «Матрица рисков (тепловая карта)»
- Сетка 5×5: ось X — Вероятность (1–5), ось Y — Влияние (5→1 сверху вниз).
- Каждая ячейка закрашена в один из **5 цветов** по итоговому баллу (Вероятность × Влияние):
  - 1–4 — зелёный (Незначительный);
  - 5–9 — светло-зелёный (Низкий);
  - 10–14 — жёлтый (Средний);
  - 15–19 — оранжевый (Высокий);
  - 20–25 — красный (Критический).
- В каждой ячейке — автоматический подсчёт количества рисков (формула COUNTIFS по данным Реестра рисков), без ручного ввода.
- Легенда цветовой шкалы с рекомендуемым реагированием для каждой зоны.
- Сводная таблица «Итого рисков по зонам» с формулами COUNTIF.

## 5. Нефункциональные требования

- **Формат файла:** .xlsx, совместимый с Excel и LibreOffice.
- **Формулы, а не хардкод:** все расчётные значения (уровень риска, зоны, подсчёты в матрице, связи между листами) реализованы формулами, обновляются динамически при изменении данных.
- **Ноль ошибок формул:** финальный файл проверен пересчётом (LibreOffice) — 320 формул, 0 ошибок (#REF!, #DIV/0!, #VALUE!, #N/A, #NAME?).
- **Единый шрифт:** Arial по всей книге.
- **Цветовая шкала:** ровно 5 цветов для классификации уровня риска, единообразно применяется на листах «Справочники», «Реестр рисков» и «Матрица рисков».
- **Расширяемость:** в Реестре рисков и Плане обработки предусмотрен запас пустых строк (40 и 30 соответственно) для дальнейшего заполнения без изменения структуры формул.

## 6. Трассируемость к методологии

| Элемент системы | ISO 31000:2018 | COSO ERM 2017 |
|---|---|---|
| Справочники (шкалы) | Установление критериев оценки риска (п. 5.3) | Strategy & Objective-Setting |
| Реестр рисков — идентификация/причины/последствия | Идентификация риска | Performance |
| Реестр рисков — вероятность/влияние/уровень | Анализ риска | Performance |
| Матрица рисков | Оценивание риска (Risk Evaluation) | Performance, Information & Reporting |
| Реестр рисков — стратегия реагирования | Обработка риска (Risk Treatment) | Performance |
| План обработки рисков | Обработка риска (Risk Treatment) | Performance |
| Дата следующего пересмотра, статус мероприятий | Мониторинг и пересмотр | Review & Revision |
| Владелец риска / Ответственный | Коммуникация и консультирование | Governance & Culture |

## 7. Результат (Deliverable)

Файл `Система_управления_рисками.xlsx`, содержащий 5 листов:
1. Инструкция
2. Справочники
3. Реестр рисков
4. План обработки рисков
5. Матрица рисков

Статус: реализовано, формулы пересчитаны, ошибок не обнаружено.

## 8. Дальнейшие шаги (не входит в v1.0)

- Настройка шкал вероятности/влияния под специфику конкретной отрасли/организации.
- Интеграция с внешними источниками данных (например, автоматическая загрузка инцидентов).
- Добавление истории изменений уровня риска (тренд) для целей Review & Revision.
- Ролевой доступ и защита листов от случайного изменения формул.
